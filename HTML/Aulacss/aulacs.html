O que é CSS e como funciona a sintaxe?

CSS (Cascading Style Sheets) é a linguagem que diz ao navegador como os elementos HTML devem parecer.

Sem CSS, toda página seria texto preto em fundo branco. O CSS separa o conteúdo (HTML) da aparência (CSS) — uma boa prática fundamental do desenvolvimento web.

/* Sintaxe básica do CSS: */

seletor {
  propriedade: valor;
  propriedade: valor;
}

/* Exemplos reais: */
h1 { color: red; font-size: 32px; }
.titulo { background-color: #1a252f; }
#logo { letter-spacing: 3px; }
✅ Regra de ouro da sintaxe: cada declaração termina com ponto e vírgula (;). Esquecer o ; é um dos erros mais comuns — e o CSS seguinte pode quebrar silenciosamente.
⚠️ CSS é "silencioso" no erro: se você errar uma propriedade ou valor, o navegador simplesmente ignora aquela linha sem avisar. Sempre abra o DevTools (F12) para ver o que foi aplicado e o que foi ignorado.

Seletores — Como o CSS encontra os elementos

O seletor diz ao CSS quem vai receber o estilo. Cada tipo tem um alcance diferente.

1. Seletor de tag — afeta TODOS os elementos daquele tipo

p { background-color: #d5f5e3; padding: 8px; }
/* Todos os <p> da página ficam verdinhos */

Eu sou um parágrafo — recebi o estilo!

Este também é um <p> — também recebeu!

Todo <p> da página é afetado.

2. Seletor de classe (.nome) — afeta quem tiver aquela class=

.destaque-aula { background-color: #fdebd0; font-weight: bold; }
/* Só elementos com class="destaque-aula" */

Parágrafos abaixo — só o com a classe é estilizado:

Parágrafo comum — sem classe, sem estilo especial.

Parágrafo com class="destaque-aula" — estilizado!

Outro parágrafo comum — também sem estilo.

3. Seletor de ID (#nome) — afeta UM único elemento

#unico-demo { background-color: #d6eaf8; border: 2px solid #2980b9; }
/* Só o elemento com id="unico-demo" — único na página */
div sem ID — nenhum estilo especial
div com id="unico-demo" — estilizado de forma única!

⚠️ IDs devem ser únicos — só pode existir um por página! Use classes para estilo (reutilizável). Use IDs para âncoras e JavaScript.

4. Seletor descendente — "pai filho" afeta o filho DENTRO do pai

.container-desc strong { color: #e74c3c; }
/* Só o <strong> dentro de .container-desc fica vermelho */

Dentro de .container-desc: este strong fica vermelho pelo seletor descendente.

Fora de .container-desc: este strong não é afetado — permanece normal.

5. Seletor de atributo — [atributo="valor"]

input[type="text"] { border: 3px solid #27ae60; } /* verde */
input[type="email"] { border: 3px solid #e74c3c; } /* vermelho */

Mesma tag <input>, estilos diferentes pelo atributo type:

✅ Muito útil para formulários: estilize campos específicos sem precisar adicionar classes em cada um.

6. Múltiplos seletores com vírgula — aplica as mesmas regras para todos

h1, h2, h3 { color: #2c3e50; } /* Evita repetição */
input[type="text"], input[type="email"], select { width: 100%; }
✅ Separe seletores com vírgula para aplicar as mesmas propriedades a vários elementos de uma vez — menos código, mais organização.

Especificidade e Cascata — Quem vence o conflito?

Quando duas regras tentam estilizar o mesmo elemento, o CSS tem critérios para decidir qual prevalece.

Pense na especificidade como uma pontuação. Quem tem mais pontos, ganha. Se empatarem, vence o que vem por último no arquivo CSS (a cascata).

p { color: gray; } /* Tag: baixa especificidade (0,0,1) */
.classe { color: blue; } /* Classe: média (0,1,0) */
#id { color: red; } /* ID: alta (1,0,0) */
style="" { color: green; } /* Inline: máxima (1,0,0,0) */
Seletor de tag — menor especificidade (cinza)
Seletor de classe — especificidade média (azul)
Seletor de ID — especificidade alta (vermelho)
Inline style="" — especificidade máxima (verde)
⚠️ Evite !important — ele força uma regra a ganhar de tudo, mas quebra a lógica da cascata e dificulta a manutenção do código. Se você precisar de !important, geralmente é sinal de que a estrutura dos seletores precisa ser revisada.

Cores — As 4 formas de escrever

CSS aceita cores de várias formas. Cada uma tem seus usos.

/* 1. Nome (só para cores básicas) */
color: tomato;

/* 2. Hexadecimal — o mais usado em design */
color: #e74c3c; /* #RRGGBB — R=vermelho G=verde B=azul em hex */
color: #e74; /* Atalho: #RGB (equivale a #ee7744) */

/* 3. RGB — valores de 0 a 255 */
color: rgb(231, 76, 60);

/* 4. RGBA — RGB + Alpha (transparência: 0=invisível, 1=sólido) */
color: rgba(231, 76, 60, 0.5); /* 50% de opacidade */

Propriedades de Texto

CSS tem um conjunto rico de propriedades para controlar como o texto aparece.

font-size — Tamanho da fonte

font-size: 12px; /* pequeno */
font-size: 18px; /* médio */
font-size: 28px; /* grande */

font-size: 12px — texto pequeno

font-size: 18px — texto médio

font-size: 28px — texto grande

font-weight — Espessura da fonte

font-weight: normal; /* igual a 400 */
font-weight: bold; /* igual a 700 */
font-weight: 900; /* mais pesado (depende da fonte) */

font-weight: normal — texto regular

font-weight: bold — texto em negrito

font-weight: 900 — texto extra bold

font-style — Itálico

font-style: italic; /* itálico */
font-style: normal; /* cancela itálico (útil em herança) */

font-style: italic — texto em itálico

font-style: normal — texto normal

text-align — Alinhamento do texto

text-align: left; /* padrão */
text-align: center;
text-align: right;

text-align: left — alinhado à esquerda (padrão)

text-align: center — centralizado

text-align: right — alinhado à direita

text-decoration — Sublinhado e outros efeitos

text-decoration: underline; /* sublinhado */
text-decoration: line-through; /* tachado */
text-decoration: none; /* remove sublinhado (links!) */

text-decoration: underline — sublinhado

text-decoration: line-through — texto tachado

text-decoration: none — remove o sublinhado de links

text-transform — Capitalização sem alterar o HTML

text-transform: uppercase; /* TUDO MAIÚSCULO */
text-transform: lowercase; /* tudo minúsculo */
text-transform: capitalize; /* Primeira Letra Maiúscula */

text-transform: uppercase — assim fica

text-transform: LOWERCASE — ASSIM FICA

text-transform: capitalize — assim fica

letter-spacing e word-spacing — Espaçamento entre letras e palavras

letter-spacing: 6px; /* espaço entre letras */
word-spacing: 12px; /* espaço entre palavras */

letter-spacing: 6px — espaçado assim

word-spacing: 12px — palavras afastadas

line-height — Altura de linha (essencial para leitura)

line-height: 1.2; /* muito comprimido */
line-height: 1.6; /* confortável para leitura (recomendado) */
line-height: 2.2; /* muito espaçado */

line-height: 1.2 — Este texto está com espaçamento muito comprimido entre as linhas. Fica difícil de ler quando tem mais de uma linha de conteúdo.

line-height: 1.6 — Este texto está com espaçamento confortável. É o valor ideal para corpo de texto em blogs, artigos e sites em geral.

line-height: 2.2 — Este texto está muito espaçado. Pode ser usado pontualmente, mas cansa em textos longos.

text-shadow — Sombra no texto

/* deslocamento-X | deslocamento-Y | desfoque | cor */
text-shadow: 2px 2px 5px rgba(0,0,0,0.5); /* suave */
text-shadow: 3px 3px 0px #e74c3c; /* nítida colorida */

Sombra suave com desfoque

Sombra nítida e colorida

Box Model — O modelo de caixa do CSS

Todo elemento HTML é uma caixa retangular composta por 4 camadas. Entender isso resolve 90% dos problemas de layout.

MARGIN (espaço externo — fora da caixa)
BORDER (a borda visível da caixa)
PADDING (espaço interno — entre borda e conteúdo)
CONTENT
(texto, imagem, etc.)
✅ Regra mental: margin afasta os elementos uns dos outros. Padding cria espaço respiração dentro da caixa.

margin — Espaço externo (entre elementos)

margin: 4px; /* pequena */
margin: 20px; /* média */
margin: 40px; /* grande */
margin: 4px
margin: 20px
margin: 40px

padding — Espaço interno (dentro da caixa)

padding: 4px; /* apertado */
padding: 16px; /* confortável */
padding: 32px; /* espaçoso */
padding: 4px
padding: 16px
padding: 32px

Shorthand de margin e padding — 1, 2 ou 4 valores

padding: 20px; /* 1 valor: todos os lados iguais */
padding: 8px 40px; /* 2 valores: vertical | horizontal */
padding: 4px 20px 12px 40px; /* 4 valores: topo | direita | baixo | esquerda (horário) */
padding: 20px — todos os lados iguais
padding: 8px 40px — pouco em cima, muito nas laterais
padding: 4px 20px 12px 40px — cada lado diferente

border — A borda visível da caixa

/* Shorthand: espessura | estilo | cor */
border: 3px solid #e74c3c; /* linha sólida */
border: 3px dashed #3498db; /* tracejada */
border: 3px dotted #27ae60; /* pontilhada */
border-left: 8px solid #9b59b6; /* só à esquerda */
border-radius: 20px; /* arredonda os cantos */
border: 3px solid — linha sólida
border: 3px dashed — tracejada
border: 3px dotted — pontilhada
border: 6px double — dupla
border-left: 8px solid — só à esquerda (padrão de citação)
border + border-radius: 20px — bordas arredondadas

width e height — Largura e altura

width: 200px; /* fixo */
width: 60%; /* relativo ao pai */
max-width: 300px; /* nunca passa disso */
height: 80px; /* altura fixa */
width: 200px — sempre 200px
width: 60% — 60% do pai
max-width: 300px; width: 100% — cresce até 300px
height: 80px — altura fixa

box-sizing — O problema que o reset resolve

/* SEM border-box: padding AUMENTA o tamanho total */
.caixa { box-sizing: content-box; width: 200px; padding: 20px; } /* Resultado: 200 + 20 + 20 + 5 + 5 = 250px no total! */

/* COM border-box: padding está INCLUÍDO no width */
.caixa { box-sizing: border-box; width: 200px; padding: 20px; } /* Resultado: 200px no total — previsível! */
content-box (padrão antigo)
width: 200px + padding: 20px + border: 5px
Total real: 250px 😱
border-box (nosso reset)
width: 200px com padding e border incluídos
Total real: 200px ✅
✅ Por isso colocamos * { box-sizing: border-box; } no reset — para que TODOS os elementos se comportem de forma previsível!

Display — Como os elementos ocupam espaço

A propriedade display controla como um elemento se comporta no layout. É uma das mais importantes do CSS.

inline — Fica na mesma linha, não aceita width/height/margin vertical

/* <span>, <a>, <strong> são inline por padrão */
span { display: inline; } /* margin-top é ignorado! */
Span 1 Span 2 Span 3 — todos na mesma linha, margin-top ignorada

block — Ocupa a linha inteira, aceita width/height/margin

/* <div>, <p>, <h1> são block por padrão */
div { display: block; width: 200px; } /* width funciona! */
Block 1
Block 2 — cada um em sua própria linha
Block 3 — width: 200px funciona

inline-block — O melhor dos dois mundos

/* Fica na mesma linha MAS aceita padding, margin e dimensões */
.botao { display: inline-block; padding: 12px 20px; }
/* Usado em links estilizados como botão! */
Botão A Botão B Botão C — lado a lado, com padding funcionando

none — Esconde o elemento completamente (some do layout)

.escondido { display: none; } /* elemento não ocupa espaço */

Parágrafo visível — normal.

Próximo parágrafo — o anterior sumiu do layout completamente.

⚠️ display: none remove o elemento do layout (não ocupa espaço). visibility: hidden esconde mas mantém o espaço. São coisas diferentes!

Pseudo-classes — Estilo baseado em estados

Pseudo-classes aplicam estilos quando o elemento está em um determinado estado — o mouse está sobre ele, o campo está selecionado, etc.

/* Sintaxe: seletor:pseudo-classe { ... } */
a:hover { color: red; } /* mouse em cima */
input:focus { border-color: blue; } /* campo ativo */
a:visited{ color: purple; } /* link já visitado */
li:first-child { color: red; } /* primeiro filho */
li:nth-child(even) { background: #eee; } /* linhas pares */

:hover — Passe o mouse sobre os botões

Cor muda no hover Cor diferente Fica transparente Sobe com sombra

:focus — Clique no campo e veja o efeito

input:focus {
  border-color: #3498db;
  outline: none; /* remove borda padrão do navegador */
  box-shadow: 0 0 0 3px rgba(52,152,219,0.2); /* halo azul */
}

:nth-child — Estilize linhas alternadas (zebra) — bônus!

li:nth-child(odd) { background: #eaf4fb; } /* ímpares */
li:nth-child(even) { background: #d6eaf8; } /* pares */
li:first-child { border-top: 3px solid #3498db; }
li:last-child { border-bottom: 3px solid #3498db; }

  • Item 2 — :nth-child(even)
  • Item 3 — :nth-child(odd)
